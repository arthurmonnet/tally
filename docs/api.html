PulseBar Push API

Receive your daily developer stats on any server via a simple JSON webhook.

PulseBar can push a daily summary of your activity to a remote HTTP endpoint. This page documents the payload format, authentication, and expected server behavior so you can build dashboards, team feeds, or personal analytics on top of your data.

How it works

PulseBar collects stats locally (keystrokes, clicks, app usage, git activity, etc.) and periodically sends a POST request with a JSON payload to the endpoint you configure in the app.

Authentication

Every request includes a Bearer token in the Authorization header. Set your token in the PulseBar settings panel — it is stored securely in the macOS Keychain.

Authorization: Bearer <your-api-token>

Your server should validate this token and reject requests with a missing or invalid token with a 401 status.

Endpoint Setup

Configure your endpoint URL in PulseBar → Pulse API settings.

HTTPS Required

The endpoint must use HTTPS. The only exception is localhost / 127.0.0.1 for local development.

Request Format

POST {your-endpoint-url}

Headers

HeaderValue
Content-Typeapplication/json
AuthorizationBearer <token>

Request timeout is 30 seconds.

Each entry in the top_apps array:

Expected Response

Your server should return an appropriate HTTP status code:

StatusMeaning
200-299Push accepted. PulseBar records success.
401Invalid or missing API token.
4xx / 5xxPush failed. PulseBar will retry once after 5 seconds.

The response body is not inspected — only the status code matters.

Push Frequency

Choose how often PulseBar sends stats:

OptionInterval
Every 30 min1,800 s
Every hour3,600 s
Every 3 hours10,800 s
Manual onlyNo auto-push; use the Push Now button.

PulseBar also pushes automatically when the app quits and when the Mac wakes from sleep (if the interval has elapsed).

Retry & Scheduling

  • If a push fails, PulseBar retries once after a 5-second delay.
  • If both attempts fail, the error is shown in the status area.
  • On wake-from-sleep, a push fires immediately if the configured interval has passed.
  • A final push is attempted on app quit (10 s timeout).

Minimal Server (Node.js)

// server.js
const express = require("express");
const app = express();

app.use(express.json());

app.post("/api/pulse", (req, res) => {
  const token = req.headers.authorization?.replace("Bearer ", "");

  if (token !== process.env.PULSE_TOKEN) {
    return res.status(401).json({ error: "Unauthorized" });
  }

  console.log(`[${req.body.date}]`, `${req.body.keystrokes} keystrokes`);
  // Store req.body in your database

  res.status(200).json({ ok: true });
});

app.listen(3000);
